# greentic-demo

Minimal Rust workspace for collecting many Greentic demos as independent crates.

## Layout

- `crates/<demo_name>/`: source for each demo crate.
- `demos/<name>.gtpack`: built demo application packs generated by the packaging script or GitHub Actions.
- `demos/<demo_name>.gtbundle`: packaged demo bundles generated by the packaging script or GitHub Actions.
- `ci/local_check.sh`: local validation for the workspace and demo packaging flow.

Published outputs:
- app packs to `ghcr.io/<owner>/packs/apps/<pack_name>:<tag>`
- demo packs to `ghcr.io/<owner>/packs/demos/<pack_name>:<tag>`
- demo bundles to `ghcr.io/<owner>/bundles/<demo_name>:<tag>`
- tagged demo pack and bundle release assets under GitHub Releases

## Add a Demo

1. Create a crate in `crates/<demo_name>`.
2. Add a standard `Cargo.toml` manifest for that demo.
3. Run `ci/local_check.sh` to verify the workspace and bundle output.

## Package Demos

Run:

```bash
scripts/package_demos.sh
```

The packaging flow now:
- builds real demo `.gtpack` artifacts via `greentic-pack wizard apply`
- replays the demo bundle answer files with those local `.gtpack` artifacts plus OCI extension refs
- emits `demos/*.gtbundle` and `demos/*.gtpack`

## Publish Demos

The GitHub Actions publish workflow:
- builds demo `.gtpack` artifacts and bundle `.gtbundle` artifacts from wizard answer documents
- publishes demo packs to `ghcr.io/<owner>/packs/demos/<pack_name>:<sha>`
- publishes demo bundles to `ghcr.io/<owner>/bundles/<demo_name>:<sha>`
- publishes `:latest` on `main` / `master`
- publishes `:<version>` on versioned releases
- attaches `demos/*.gtpack` and `demos/*.gtbundle` to GitHub Releases
